# 云端真实语音与导出验收

> 状态：人工验收阻塞。自动测试和 mock 不能替代真实麦克风、Chrome/Edge 与长时验收。

## 执行入口

1. 运行 `python main.py`。
2. 在 `http://127.0.0.1:8000/diagnostic.html` 完成环境预检。
3. 进入 `http://127.0.0.1:8000/?acceptance=1`。
4. 分别在 Chrome 与 Edge 中填写环境信息，按验收台顺序说出 20 条云端指令。
5. 核对自动记录的实际转写、执行结果、本地耗时、端到端耗时和错误分类；必要时人工修正通过/失败。
6. 分别导出 JSON 与 Markdown 报告，并保留导出文件作为人工验收证据。

验收数据仅保存在当前浏览器本地，不保存音频；可从验收台明确清除。

## 20 条标准指令

验收台内置并逐条记录以下云端指令：

1. 画一个红色圆形
2. 画一个蓝色矩形放在右边
3. 画一个文字，写上“你好”放在中央
4. 画一个房子，然后移动到右边
5. 画一个雪人，然后复制它
6. 画一排三个矩形，然后顶部对齐
7. 选择所有圆形
8. 选择所有图形
9. 把它改成绿色
10. 把它向下移动五十
11. 把它放大两倍
12. 把它旋转四十五
13. 顶部对齐
14. 横向均匀分布
15. 组合这些图形
16. 选择组合一
17. 撤销
18. 重做
19. 保存为 SVG
20. 保存为 PNG

## 通过门槛

- Chrome 与 Edge 各自至少 `18/20` 条云端标准指令正确执行。
- 两个浏览器均通过首次授权、TTS 暂停恢复、云端失败重试、30 分钟连续使用和 SVG/PNG 导出。
- 本地规则最终文本到执行完成目标低于 `100ms`。
- 简单语音指令端到端延迟目标低于 `1s`。
- 在两份浏览器报告达到门槛前，候选版状态保持“人工验收阻塞”，不得标记完整发布通过。
