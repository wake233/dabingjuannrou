# 听画设计文档

## 1. 产品目标

听画面向无法或不希望在创作过程中使用鼠标键盘的用户。首版聚焦结构化矢量绘图，以确定性操作、低延迟反馈和可撤销事务为核心。用户可使用鼠标启动应用并完成首次麦克风授权；进入画板后的创作操作均有语音入口。

## 2. 架构与关键决策

应用由三个独立层次组成：

1. `static/model.js` 是纯 JavaScript 状态模型，负责动作校验、事务执行、历史记录和对象命名，不依赖 DOM。
2. `static/parser.js` 是本地中文规则解析器，负责文本归一化、中文数字、同义词、分句和上下文引用。
3. `static/app.js` 负责云端语音采集、SVG 渲染、语音反馈、模型回退和文件导出。

`main.py` 使用 Python 标准库托管静态文件，并提供可选的 OpenAI 兼容模型代理。模型密钥不会进入浏览器。

### 准确性与容错

- 本地规则覆盖高频指令，保证同一句话稳定地产生相同动作。
- 标点及“然后、再、并且”等连接词用于拆解复杂指令。
- 支持中文数字、部分同义词，以及“它、刚才两个、这些图形、矩形一”等引用。
- 带置信度的文本入口低于 `0.45` 时不执行未知指令；未知指令回退至模型，模型不可用时提示改用标准指令。
- 清空画布需要在 8 秒内再次说“确认”，避免不可逆误操作。

### 响应延迟

- 规则解析、状态更新和 SVG 渲染均在浏览器本地完成，并显示从最终识别文本到执行完成的耗时，目标低于 100ms。
- 模型回退开始时立即播报“正在理解”；网络耗时不阻塞页面。
- 语音播报期间暂停识别，播报结束后自动恢复，避免应用识别自己的声音。

### 复杂指令与事务

一段语音最多拆成 20 个标准动作。所有修改动作先在状态副本中执行，任何动作失败则整段指令不改变画布；成功后只生成一条历史记录，因此一次“撤销”可回退整段复杂指令。

## 4. 标准动作接口

允许的动作类型为 `create`、`select`、`update`、`move`、`align`、`distribute`、`duplicate`、`delete`、`group`、`ungroup`、`history`、`canvas`、`export`、`help`、`status`。

对象目标可以是 `selected`、`last`、`lastTwo`、`all`、对象 ID 或自动名称。模型动作在服务端和浏览器端均按动作类型校验必需字段、字段类型、允许枚举和值域；不支持代码执行、任意属性写入或绕过清空确认。

## 5. 测试与验收结果

- Node 内置测试覆盖中文数字、文本归一化、复杂分句、上下文引用、动作校验、事务回滚、撤销重做、布局、组合、删除，以及前端渲染、确认、低置信度、模型回退和导出流程。
- Python `unittest` 覆盖模型未配置、有效响应、非法 JSON、非法动作和超时。
- 自动测试命令与结果：
  - `node --test tests/*.test.js`：90 项通过。
  - `python -m unittest discover -s tests -p "test_*.py" -v`：25 项通过。

## 6. 未完成项与原因

- 未实现自由手绘、图片导入、项目文件恢复和生成式图像：这些能力不属于首版结构化矢量绘图范围。
- 未实现从系统启动开始的完全零鼠标键盘：浏览器首次麦克风授权受安全策略控制，网页无法自行确认。
- 未提供离线语音识别：当前仅保留云端 STT 正式链路，以降低安装体积并保证中文识别质量。
- 未对真实麦克风识别准确率建立自动化测试：云端转写结果依赖设备、网络和环境噪声，需要人工语音验收。

## 7. Git 提交要求

- 每次操作完成并完成必要验证后，都必须创建一次 Git commit。
- 提交时只包含本次操作产生的改动，不得混入用户已有或其他无关的未提交改动。
