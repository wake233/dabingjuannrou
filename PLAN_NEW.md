# 语音驱动绘本场景创作 V1

## Summary

将“听画”从基础图形指令工具升级为可持续修改的绘本场景创作系统：

`用户描述 → AI 场景规划 → 本地 SVG 模板构图 → 语义实体画布 → 继续语音修改`

首版采用通用混合实体库、单一绘本风格和整体 SVG 实体。完整场景默认追加到当前画布；未知实体忽略并明确反馈。普通基础绘图指令继续使用现有本地低延迟链路。

## Core Design

- 新增场景语义层，记录主题、情绪、构图说明、场景摘要、实体和被忽略内容。
- 画布同时支持现有基础图形与语义实体；人物、猫、雨、树等实体具有稳定中文名称，可被后续指令引用。
- 整体 SVG 实体不可逐部件选择，但暴露受控参数，例如颜色、姿态、朝向、数量和密度。
- 本地模板负责最终 SVG，不允许 LLM 输出原始 SVG、路径或任意属性。
- AI 只负责选择模板、规划位置层次、设置参数和解释构图。
- 场景生成与修改均作为单次原子事务提交，可一次撤销。

首批模板固定为：

- 人物与动物：人物、猫
- 道具与建筑：伞、路灯、屋顶、建筑剪影
- 天气与天空：雨、云、太阳、月亮、星空
- 自然与环境：树、山、花丛、河流、草地、街道、水洼

唯一风格为 `storybook`，使用低饱和调色板、柔和轮廓和绘本比例规则。

## Interfaces And Implementation

- 新增 `/api/interpret`，用于本地规则无法处理的表达；服务端返回带 `kind` 的受校验联合结果：
  - `actions`：现有标准动作。
  - `scene_plan`：新增或构建场景。
  - `scene_revision`：修改已有语义实体或场景参数。
- 场景规划包含主题、情绪、构图说明、摘要、忽略项，以及最多 20 个实体实例。每个实例只能引用白名单模板、参数、语义名称、角色、边界和层次。
- 向状态模型增加 `scene` 元数据和 `entity` 对象类型；实体保存模板 ID、语义名称、位置尺寸、旋转、透明度和受控参数。
- 新增受校验动作 `entity_create`、`entity_update` 和 `scene_update`；现有移动、缩放、删除、置顶和选择动作同时支持实体。
- 项目格式升级到版本 2；加载版本 1 项目时自动补充空场景元数据，保持现有作品兼容。
- 新建本地模板注册表。模板使用可信 JavaScript 创建 SVG DOM；模型只能调整模板明确声明的参数。
- 本地规则增加常见实体修改：
  - “把伞改成红色”
  - “女人往左一点”
  - “删除那只猫”
- 雨量、姿态、朝向、情绪等复杂修改交给 `/api/interpret`。
- 非空画布上的场景生成默认追加；规划请求携带现有实体名称、参数和边界，AI 应将新内容放入可用区域。
- 生成期间立即提示“正在构思”；生成后侧栏展示创作说明、实体列表和被忽略内容，不增加生成前确认。
- 若规划结果没有任何受支持实体，则不修改画布并反馈无法表达的内容。
- SVG、PNG 和项目导出必须完整保留语义实体的渲染结果与可编辑数据。

## Test Plan

- 校验服务端和浏览器端均拒绝未知模板、非法参数、原始 SVG、非法场景规划和超过 20 个实体的结果。
- 验证场景生成、修改和失败回滚均只产生一条撤销记录。
- 验证版本 1 项目可迁移加载，版本 2 项目可保存并恢复实体参数和场景说明。
- 验证实体可按中文名称选择、移动、缩放、改色、删除和修改模板参数。
- 验证未知实体被忽略、显示在创作说明中，并通过语音反馈告知用户。
- 验证追加生成会参考已有实体边界，避免默认全部重叠于画布中央。
- 验证实体正确渲染并进入 SVG、PNG 导出。
- 完成三组端到端验收：
  1. 雨中人物：生成后将伞改红、人物左移、雨量加大。
  2. 月夜屋顶猫：修改猫朝向、放大月亮、删除云。
  3. 春日山野：增加花丛、移动太阳、调整河流颜色。
- 运行完整 Node、Python 和 smoke 测试；验证通过后仅提交本次功能相关改动并创建独立 Git commit。

## Assumptions

- 首版不接入文生图、外部 SVG 素材库或自由局部重生成。
- 实体内部部件不可单独选择。
- 绘本风格暂不支持运行时切换。
- AI 服务不可用时，现有基础绘图能力继续工作；艺术场景生成明确提示不可用。
- 被忽略实体不会使用文字占位符，也不会导致其余场景生成失败。
