# 听画艺术创作升级路线

## 1. 总结

将听画升级为面向单幅叙事插画的“语音导演式创作工作室”。

核心流程：

```text
表达题材与意图
  → 选择风格
  → 查看三张构图小稿
  → 选择或混合方案
  → 锁定满意部分
  → 多轮审美精修
  → 自动生成纹理
  → 导出
```

第一阶段完整打通绘本风格纵向链路，再扩展木刻与水墨。继续保持语音优先、受约束矢量、事务撤销和安全校验。

艺术级版本的核心验收标准是：

- 同一题材由不同用户创作时，可以得到明显不同但都成立的作品。
- 用户可以表达并修改焦点、氛围、节奏、光线、留白和局部造型。
- 作品经过多轮修改后能够实质改善，而不只是移动对象或更换颜色。

## 2. 核心改造

### 2.1 创作状态

项目格式升级至版本 3，新增以下结构化创作状态：

- `intent`：叙事、情绪、焦点、节奏与象征意图。
- `composition`：视觉动线、负空间、前中后景、光源与明暗关系。
- `artDirection`：风格、色板、线条语言、形状语言和纹理提示词。
- `locks`：锁定构图、配色、光线或指定实体；后续修改必须保持锁定内容。
- `drafts`：三张待选择的构图小稿、选中方案及当前创作阶段。

实体从固定整体 SVG 模板升级为“语义实体 + 参数化部件图”：

- 人物支持头部、躯干、四肢、服装、表情和姿态变化。
- 树木、建筑和主要环境实体支持轮廓、比例与细节变体。
- 部件不作为顶层对象，不占用场景实体数量限制。
- 三种风格共享语义实体和构图，但由独立渲染器解释造型与材料。

### 2.2 创作流程

- 用户在生成小稿前选择 `storybook`、`woodcut` 或 `ink` 风格。
- 系统生成三张符合目标风格的低细节矢量构图小稿。
- 三张小稿必须在焦点位置、视觉动线、尺度关系和负空间方面明显不同。
- 用户可以选择、混合或重新生成小稿；选定方案后作为单一事务进入正式画布。
- 支持“更孤独”“加强风感”“右侧留白更多”等审美意图修改。
- 支持“锁定人物”“保持构图”“不要改变配色”等语音锁定操作。
- 第一阶段只完整实现绘本渲染器；木刻与水墨随后基于同一语义结构实现独立渲染规则。

### 2.3 三种风格

- `storybook` 层叠绘本：柔和轮廓、低饱和色彩、细腻叙事和明确前中后景。
- `woodcut` 木刻版画：高对比明暗块、方向性刻线、有限色板和强烈形状节奏。
- `ink` 水墨：浓淡层次、留白规则、飞白和受控墨迹纹理。

三种风格不能仅通过更换色板实现。每种风格必须具备独立的造型、构图倾向和渲染规则。

### 2.4 云端纹理

- 用户选定小稿后自动请求云端生成纹理，同时先展示完整矢量结果。
- 普通编辑不会自动重新生成纹理；用户可以显式要求重新生成。
- 原始纹理保存在 IndexedDB，本地工程保存提示词、模型信息和缓存键。
- 缓存丢失时展示无纹理矢量版本，并允许用户重新生成。
- SVG、PNG 导出固化当前可用纹理。
- 纹理失败不得阻塞绘制、编辑、撤销、工程恢复或导出。
- 服务端限制纹理类型、尺寸和文件大小，不接受外部 URL 或模型生成的 SVG。

## 3. 接口与动作能力

新增受约束服务接口：

- `/api/compose-drafts`：根据题材、风格和现有上下文返回三套结构化小稿。
- `/api/refine-artwork`：根据审美意图和锁定状态返回受校验的场景修改。
- `/api/generate-texture`：返回受尺寸限制的云端纹理及模型元数据。

新增动作能力：

- 选择、混合或重新生成小稿。
- 更新审美意图与艺术指导。
- 锁定或解锁场景字段及实体。
- 应用、移除或重新生成纹理。

所有新增结果继续在服务端和浏览器端进行双重校验，不允许模型输出原始 SVG、路径、脚本、外部 URL 或任意属性。

## 4. 实施阶段

### 阶段一：创作流程基础

- 建立版本 3 创作状态和版本 2 工程迁移。
- 实现审美意图、锁定机制和三张构图小稿流程。
- 建立约 12 个精选叙事题材的作品验收集。

### 阶段二：完整绘本纵向切片

- 完成绘本风格参数化实体和独立渲染器。
- 完成审美精修、方案混合与锁定约束。
- 接入云端纹理生成、本地缓存、工程恢复与导出。
- 绘本纵向切片达到作品集验收标准后，才开始另外两种风格。

### 阶段三：木刻版画

- 实现高对比明暗块、方向性刻线和有限色板。
- 根据木刻风格特性调整小稿构图与负空间策略。
- 验证从同一语义场景切换至木刻时仍保持叙事关系。

### 阶段四：水墨

- 实现浓淡层次、留白规则、飞白和受控墨迹纹理。
- 根据水墨风格特性调整空间、节奏与视觉焦点。
- 验证水墨纹理不会破坏主体可编辑性。

### 阶段五：收口

- 完善跨风格切换、性能和工程兼容。
- 完成真实语音创作验收和精选作品集评审。
- 每个阶段完成必要验证后创建独立 Git commit。

## 5. 测试与验收计划

- 验证三张小稿结构合法、构图明显不同，并可选择、混合、撤销和恢复。
- 验证审美意图修改有效，所有锁定字段和实体保持不变。
- 验证版本 2 工程迁移、版本 3 保存、创作阶段恢复和纹理缓存缺失回退。
- 验证纹理接口拒绝非法格式、超限数据、外部 URL 和任意 SVG。
- 验证纹理失败不影响矢量编辑、撤销、项目恢复和导出。
- 为三个渲染器建立确定性 SVG 结构测试和人工视觉验收。
- 使用约 12 个精选叙事题材建立作品集，按构图、视觉层级、叙事、风格一致性和完成度评分。
- 验证同一题材可通过不同审美意图和小稿选择形成明显不同的有效作品。
- 验证用户可以通过语音准确完成焦点、氛围、节奏、光线、留白和锁定修改。

## 6. 假设与默认选择

- 首个艺术级里程碑聚焦单幅叙事插画与绘本风格。
- 云端模型负责结构化规划和纹理，不输出或执行原始 SVG。
- 工程在原设备依靠本地缓存保持纹理原貌；跨设备允许重生成后产生差异。
- 三种风格共享语义场景，但拥有独立造型与渲染规则。
- 艺术表现力优先于开放题材覆盖率，首版围绕精选题材做到作品级质量。
- 不在本计划中实施自由手绘、无限制路径编辑或以文生图结果替代矢量主体。
