# 绘本造型与线条精修计划

## 1. 总结

优先重做 `storybook` 绘本渲染器，覆盖全部 26 个语义实体。目标是将当前的简单图标拼接效果升级为接近写实结构的细腻钢笔绘本插画。

木刻与水墨本轮保持现状。旧工程无需迁移，打开后自动使用新绘本渲染器。

视觉方向固定为：

- 接近写实的结构、姿态、透视与自然规律。
- 具有起笔、压笔、收笔和轻微手绘偏差的钢笔线条。
- 柔和渐变、局部光效、结构阴影与材质排线。
- 渐进式精绘：先快速显示主体，再补充完整细节。

## 2. 实施变更

### 2.1 钢笔线条引擎

- 新增确定性钢笔笔触模块，将中心线采样并生成封闭的可变宽 SVG 路径，不再依赖统一 `stroke-width`。
- 支持主轮廓、结构线、纹理线和氛围线四级线条；分别定义粗细、透明度、断续和扰动范围。
- 使用 `实体 ID + 部件名称` 生成固定 seed，使轻微弯曲、边缘缺口、断笔和排线每次恢复一致。
- 所有生成坐标必须有限、处于受控范围，并限制单实体节点数量，避免异常路径和性能失控。

### 2.2 全部 26 个实体精修

按结构家族重建绘本模板：

- 角色与动物：人物、猫、狗、鸟使用骨架、体块、关节、前后遮挡和姿态相关轮廓；增加面部、毛发、衣褶和羽毛结构线。
- 建筑与器物：房屋、屋顶、桥、船、长椅、自行车、路灯、伞、栅栏、建筑群使用可信透视、厚度、连接关系和材质细节。
- 自然实体：树、山、云、花丛、草地使用不规则轮廓、生长方向、疏密变化和前后穿插。
- 场景与光效：雨、河流、街道、水洼、太阳、月亮、星空使用方向场、反光、波纹、投影和局部高光。

每个实体必须包含独立、可辨识的外轮廓，禁止不同实体复用同一通用剪影。

### 2.3 色彩与光线

- 每个绘本实体使用受信任的本地线性或径向渐变，表现主体明暗转折。
- 使用少量半透明高光、接触阴影和投影建立体积；光效不能遮盖结构线。
- 近景实体增加材质排线，中景降低纹理密度，远景只保留主要轮廓和明暗块。
- 渐变与滤镜 ID 使用渲染命名空间，避免主画布、语音预览和导出之间发生 ID 冲突。

### 2.4 渐进式精绘与导出

- `renderArtworkEntity` 增加只读渲染选项：`quality: "base" | "full"` 和唯一 `namespace`。
- 主画布首次渲染使用 `base`，包含外轮廓、主要色块和关键阴影。
- 下一动画帧使用 `full` 替换为结构线、材质、渐变和完整光效；新渲染开始时取消旧精绘任务。
- 语音中间预览仅使用 `base`，避免频繁生成完整细节。
- SVG、PNG 和作品集页面始终从当前状态重新生成 `full` 质量画面，不能导出尚未完成的基础轮廓。
- 不修改工程状态、动作协议或撤销历史；渐进式替换仅属于显示层行为。

## 3. 视觉验收

新增两个真实视觉验收页面：

- **26 实体图鉴**：每个实体同时展示近景完整精绘和缩小后的可辨识效果。
- **六张标杆场景**：雨中人物、月夜屋顶猫、春日河岸、安静街道、花园里的狗、山路旅人。

移除当前自动填充完成度分数的作品集验收方式。新的验收要求：

- 26 个实体仅看轮廓时仍可互相区分。
- 人物和动物姿态不存在明显关节、遮挡或比例错误。
- 建筑与器物连接关系和透视可信。
- 每个核心实体至少包含三级可见线条粗细。
- 近景、中景和远景的细节密度明显不同。
- 六张标杆场景均通过人工评审：结构可信、线条细腻、光影统一、没有图标拼接感。
- 浏览器视觉证据更新后，才能标记精修完成。

## 4. 测试计划

- 单元测试验证钢笔路径确定性、可变宽、起收笔、受控扰动、有限坐标和节点上限。
- 模板测试验证全部 26 个实体具有独立语义轮廓、完整与基础两种质量，并包含线条层级和光影结构。
- 应用测试验证基础渲染立即出现、完整精绘下一帧替换、过期任务被取消。
- 验证预览只生成基础质量，SVG、PNG 始终包含完整精绘且不包含选择高亮。
- 验证旧版工程加载后数据不变，但自动使用新绘本渲染器。
- 性能目标：20 个实体基础渲染低于 `100ms`；完整精绘在基础画面出现后 `500ms` 内完成。
- 每阶段运行 Node、Python、smoke、浏览器视觉检查和 `git diff --check`，并创建独立 Git commit。

## 5. 假设与默认选择

- 本轮只精修绘本风格，木刻与水墨渲染器不改动。
- 绘本线条风格固定，不增加新的用户动作、语音参数或工程字段。
- 旧工程视觉外观允许因新渲染器而改变。
- 所有精致效果由本地可信 SVG 渲染器生成，不允许模型输出原始路径或任意 SVG。
