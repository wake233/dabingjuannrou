# 听画

听画是一款本地运行、通过简体中文语音控制的结构化矢量绘图工具。首次启动和麦克风授权可以使用鼠标；进入画板后，创建、编辑、布局、撤销、清空与导出均可通过语音完成。

## 启动

要求 Python 3.8+。云端语音识别需要浏览器支持 `MediaRecorder`、`AudioContext`。

```powershell
python main.py
```

服务默认打开 `http://127.0.0.1:8000`。点击一次“开始聆听”会启动云端识别并请求麦克风权限；说“休息”或“停止聆听”会停止识别。若不希望自动打开浏览器：

```powershell
python main.py --no-browser
```

服务会独占监听端口，避免多个旧进程同时响应请求导致 `/api/transcribe` 随机返回 404。若提示端口已占用，请先关闭旧的听画服务。

云端语音转写依赖网络。未配置 `OPENAI_API_KEY` 或云端请求失败时，页面会提供“重试云端识别”和“停止聆听”。应用本地规则解析、绘图、工程自动保存和导出不需要额外依赖。

## 第二版识别模式

第二版普通用户界面仅提供云端识别。浏览器 Web Speech 识别和后台唤醒词已移除；真实 Vosk 离线识别尚未完成，相关模块仅作为隔离实验脚手架保留，不提供页面下载、模式切换或语音入口。

## 工程恢复

每次成功事务、撤销、重做和工程加载后都会更新本地自动保存；刷新页面会校验并恢复最近状态。损坏存档会被忽略，可说“丢弃上次工程”移除自动保存。

## 云端高准确率语音识别

应用默认使用云端音频转写，并通过静音检测自动切分每条指令。配置 OpenAI 兼容的音频转写接口后重启服务：

```powershell
# 在 .env 中填写 OPENAI_API_KEY
# 在 config.yaml 中配置服务地址和模型
python main.py
```

服务启动时自动读取项目根目录的 `.env` 和 `config.yaml`。`.env` 只保存 `OPENAI_API_KEY` 且已被 Git 忽略，可提交的密钥模板为 `.env.example`；服务地址和模型名均写在可提交的 `config.yaml` 中。音频只转发到配置的转写服务，不会保存到本地磁盘。

当前示例配置使用阿里云百炼兼容接口：语音转写模型为 `qwen3-asr-flash`，命令模型为 `qwen-plus`。实时 WebSocket 模型（名称通常包含 `realtime`）不能用于当前短音频转写请求。

云端请求失败时，页面会区分缺少配置、密钥无效、权限不足、额度不足、模型或接口不存在、网络错误、连接超时、无效响应和空转写。API 错误响应保留 `error` 文本，并提供稳定的 `errorCode` 与 `retryable` 字段。

## 引导式真实语音验收

先打开 `http://127.0.0.1:8000/diagnostic.html` 完成环境预检，再进入 `http://127.0.0.1:8000/?acceptance=1`。验收面板会复用主应用语音与绘图链路，按 20 条云端标准指令记录转写、结果和耗时，并可导出 JSON/Markdown。验收数据仅保存在当前浏览器，不保存音频。

## 示例指令

- `画一个红色圆形放在左边，再画一个蓝色矩形放在右边`
- `画一个文字，写上“你好”放在中央`
- `选中刚才两个图形，然后顶部对齐，然后整体向下移动五十`
- `选择矩形一，然后把它改成绿色`
- `把它放大两倍，然后旋转四十五`
- `选择所有图形，然后横向均匀分布`
- `撤销`、`重做`、`当前状态`、`帮助`
- `设置背景为灰色`
- `清空画布`
- `保存为 SVG`、`保存为 PNG`

每个图形会按类型自动命名，例如“矩形一”“矩形二”“圆形一”。

## 可选模型回退

规则解析器不能理解指令时，服务端可调用 OpenAI 兼容的聊天补全接口。接口地址和模型通过 `config.yaml` 的 `command_model` 配置。

密钥只保留在 Python 服务端。模型返回的动作会在服务端和浏览器内校验，每条语音最多执行 20 个动作。未配置模型时，全部标准指令仍可使用。
