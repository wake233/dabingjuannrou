# 听画

听画是一款本地运行、通过简体中文语音控制的结构化矢量绘图工具。首次启动和麦克风授权可以使用鼠标；进入画板后，创建、编辑、布局、撤销、清空与导出均可通过语音完成。
视频链接：https://www.bilibili.com/video/BV1EkJK6qEXZ/

## 绘本场景创作

听画支持把开放式中文描述交给受约束的场景规划器，再由本地可信 SVG 模板构图。画布可同时包含基础图形和具有稳定中文名称的语义实体，例如人物、动物、房屋、桥、船、街景和自然环境。模型不能输出原始 SVG、路径或任意属性，只能选择固定模板、位置、层次和受控参数。

绘本模板采用分层造型而不是简易图标：人物包含面部、发型、衣着、姿态和阴影，建筑包含门窗与灯光，天气和自然元素包含疏密、反光与前后层次。场景规划器会为常见描述主动组织前景、中景、背景、视觉焦点和协调色板。

雨中人物、公园、河岸、月夜、街景和春日山野等高频场景有本地确定性构图，即使模型暂时不可用也能生成完整画面；其他开放式场景继续使用受约束的 AI 规划器。

生成后的场景可继续说“把伞改成红色”“女人往左一点”“删除那只猫”等指令修改。复杂的雨量、姿态、朝向和情绪修改由 `/api/interpret` 返回受前后端双重校验的场景 revision。完整场景默认追加到当前画布，未知实体会被忽略并显示在创作说明中。

## 艺术导演式创作

版本 3 工程新增创作意图、构图、艺术指导、锁定、三张小稿与纹理元数据。可以先说“生成三张水墨构图小稿，雨中归人”，再说“选择第二张小稿”“混合第一和第三”“保持构图”“右侧留白更多”或“切换成木刻风格”。

绘本、木刻和水墨共享可编辑语义实体，但使用独立渲染规则：绘本强调层叠叙事，木刻使用高反差块面与方向性刻线，水墨使用浓淡、留白与飞白。选稿后会请求受约束 PNG 纹理；纹理缓存缺失或生成失败时自动回退到完整矢量版本，不影响编辑、撤销、恢复和导出。

云端纹理是可选能力。启用时，在 `config.yaml` 的 `texture_model` 中配置支持 `/images/generations` 和 `b64_json` 的 OpenAI 兼容接口，并在 `.env` 中设置 `TEXTURE_API_KEY`；未单独设置时会回退使用 `OPENAI_API_KEY`。服务只接受内联 PNG，拒绝外部 URL、SVG、超过 2048 像素或 5MB 的响应。未配置或请求失败时，作品继续使用无纹理矢量版本。

## 启动

要求 Python 3.8+。云端语音识别需要浏览器支持 `MediaRecorder`、`AudioContext`。

```powershell
python main.py
```

服务默认打开 `http://127.0.0.1:8000`。点击一次“开始聆听”会启动云端识别并请求麦克风权限；说“休息”或“停止聆听”会停止识别。若不希望自动打开浏览器：

```powershell
python main.py --no-browser
```

服务会独占监听端口，避免多个旧进程同时响应请求导致 `/api/transcribe` 随机返回 404。若提示端口已占用，请先关闭旧的听画服务。

云端语音转写依赖网络。未配置 `OPENAI_API_KEY` 或云端请求失败时，页面会提供“重试云端识别”和“停止聆听”。应用本地规则解析、绘图、工程自动保存和导出不需要额外依赖。

画布标题栏提供撤销、重做、清空和导出快捷控制。按钮与语音指令复用同一事务和历史机制；点击只显示视觉反馈，不会触发语音播报。导出菜单支持 SVG、PNG 和可继续编辑的工程文件。

## 第二版识别模式

第二版普通用户界面仅提供云端识别。浏览器 Web Speech 识别和后台唤醒词已移除；真实 Vosk 离线识别尚未完成，相关模块仅作为隔离实验脚手架保留，不提供页面下载、模式切换或语音入口。

## 工程恢复

每次成功事务、撤销、重做和工程加载后都会更新本地自动保存；刷新页面会校验并恢复最近状态。损坏存档会被忽略，可说“丢弃上次工程”移除自动保存。

## 云端高准确率语音识别

应用默认使用云端音频转写，并通过静音检测自动切分每条指令。配置 OpenAI 兼容的音频转写接口后重启服务：

```powershell
# 在 .env 中填写 OPENAI_API_KEY
# 在 config.yaml 中配置服务地址和模型
python main.py
```

服务启动时自动读取项目根目录的 `.env` 和 `config.yaml`。`.env` 保存 `OPENAI_API_KEY`，并可选保存独立的 `TEXTURE_API_KEY`；该文件已被 Git 忽略，可提交的密钥模板为 `.env.example`。服务地址和模型名均写在可提交的 `config.yaml` 中。音频只转发到配置的转写服务，不会保存到本地磁盘。

当前示例配置使用阿里云百炼兼容接口：语音转写模型为 `qwen3-asr-flash`，命令模型为 `qwen-plus`。实时 WebSocket 模型（名称通常包含 `realtime`）不能用于当前短音频转写请求。

云端请求失败时，页面会区分缺少配置、密钥无效、权限不足、额度不足、模型或接口不存在、网络错误、连接超时、无效响应和空转写。API 错误响应保留 `error` 文本，并提供稳定的 `errorCode` 与 `retryable` 字段。

持续聆听期间会监测麦克风轨道、录音器、音频上下文和页面恢复状态。非用户主动停止造成的中断会自动恢复一次；再次失败后进入手动重试状态，避免无限重启或重复录音实例。

## 引导式真实语音验收

先打开 `http://127.0.0.1:8000/diagnostic.html` 完成环境预检，再进入 `http://127.0.0.1:8000/?acceptance=1`。验收面板会复用主应用语音与绘图链路，按 20 条云端标准指令记录转写、结果、耗时、会话中断与自动恢复结果，并可导出 JSON/Markdown。验收数据仅保存在当前浏览器，不保存音频。

- `画一个红色圆形放在左边，再画一个蓝色矩形放在右边`
- `画一个文字，写上“你好”放在中央`

- `设置背景为灰色`
- `清空画布`
- `保存为 SVG`、`保存为 PNG`
- `保存项目`
- `画一个雨中人物的绘本场景`
- `画一个下雨天打伞的女人`

## 可选模型回退

规则解析器不能理解指令时，服务端可调用 OpenAI 兼容的聊天补全接口。`/api/interpret` 返回基础动作、场景规划或场景修改三种受校验结果；旧 `/api/parse` 动作接口继续兼容。接口地址和模型通过 `config.yaml` 的 `command_model` 配置。

密钥只保留在 Python 服务端。模型返回的动作会在服务端和浏览器内校验，每条语音最多执行 20 个动作。未配置模型时，全部标准指令仍可使用。
