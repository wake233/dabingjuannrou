# 听画设计文档

> 当前实现基线：2026-06-12
> 本文以当前工作区代码和实际测试结果为准，用于区分已经闭环的能力、仅完成框架的能力和后续规划。

## 1. 产品定位与当前结论

听画是一款本地运行、以简体中文语音为主要输入方式的结构化矢量绘图工具。产品核心不是自由手绘或生成式图像，而是把自然语言转换为一组受约束、可校验、可撤销的绘图动作。

当前版本已经闭环的主链路为：

1. 浏览器采集语音，默认通过 OpenAI 兼容音频转写服务获得文本；同时支持百炼 `qwen3-asr-flash` 的 OpenAI 兼容音频消息接口。
2. 文本优先由本地确定性规则解析，未知表达可回退至 OpenAI 兼容命令模型。
3. 动作在浏览器状态副本中按事务执行，成功后统一提交并渲染 SVG。
4. 高风险操作先进入 8 秒语音确认窗口。
5. 结果通过画面、状态文字和中文语音播报反馈。

当前版本已经具备完整的结构化绘图、云端语音、浏览器识别降级、唤醒词框架和自动测试覆盖。真实 Vosk 离线识别尚未闭环；现有离线模块是模型管理、音频路由和 mock 测试脚手架，不能视为可用的生产离线识别能力。

### 1.1 产品目标

| 目标 | 当前状态 | 说明 |
| --- | --- | --- |
| 确定性结构化绘图 | 已完成 | 高频指令由本地规则稳定映射为标准动作 |
| 复杂指令可撤销 | 已完成 | 一段指令最多 20 个动作，作为单一事务提交 |
| 语音优先操作 | 基本完成 | 首次授权可能需要点击；模式切换和模型下载仍主要依赖按钮 |
| 低延迟本地反馈 | 已完成框架 | 本地规则、状态更新和 SVG 渲染均在浏览器执行；真实设备指标待验收 |
| 云端识别与模型回退 | 已完成 | 密钥仅保存在 Python 服务端 |
| 完全离线语音识别 | 未完成 | Vosk WASM、模型解压加载和真实转写尚未实现 |

## 2. 系统架构

### 2.1 运行拓扑

```text
麦克风
  ├─ 云端模式：AudioContext + VAD + MediaRecorder
  │              └─ POST /api/transcribe ── OpenAI 兼容 STT / 百炼 Qwen ASR
  ├─ 浏览器模式：Web Speech API
  └─ 离线模式：AudioContext + VAD + VoskRecognizer 脚手架
                         │
                         v
语音文本 ── 本地 parser.js ── 标准动作 ── model.js 事务执行 ── SVG 渲染
               │
               └─ 解析失败时 POST /api/parse ── OpenAI 兼容命令模型
```

### 2.2 模块职责

| 模块 | 职责 | 依赖与边界 |
| --- | --- | --- |
| `static/model.js` | 状态模型、动作校验、目标解析、事务执行、历史记录、自动命名 | 纯 JavaScript，不依赖 DOM |
| `static/parser.js` | 中文数字、模糊纠错、文本归一化、分句、本地规则、复合图形拆解 | 输出标准动作，并调用浏览器侧动作校验 |
| `static/app.js` | 语音状态机、VAD、候选项评分、风险确认、预览与正式渲染、TTS、导出、模型回退 | 浏览器编排层，依赖 DOM 和浏览器媒体 API |
| `static/vosk_recognizer.js` | IndexedDB 模型缓存接口、离线识别器生命周期和 mock 注入 | 真实 Vosk WASM 初始化仍是桩代码 |
| `main.py` | 静态文件服务、配置加载、STT 代理、命令模型代理、服务端动作校验 | 仅使用 Python 标准库 |
| `static/index.html` / `static/styles.css` | 画布、状态面板、识别模式控件、预览层和视觉反馈 | 无构建步骤 |

### 2.3 状态模型

核心状态由 `DrawingEngine` 管理：

```text
objects       图形数组，数组顺序同时表示 SVG 图层顺序
selection     当前选中的对象 ID
background    画布背景色
counters      各图形类型的自动命名计数器
lastCreated   最近一次创建或复制的对象 ID
nextId        下一个对象 ID
groups        groupId 到成员 ID 数组的映射
nextGroupId   下一个组合 ID
```

每次修改动作都先作用于深拷贝状态。任意动作失败时不提交；全部成功后只生成一条撤销历史。撤销栈最多保留 50 条，执行新事务后清空重做栈。

`export`、`help`、`status` 是效果动作，不修改状态；`history` 必须单独执行。

### 2.4 语音模式与状态转换

#### 云端模式

- 默认识别模式。
- 服务端状态接口会检查云端转写配置；未配置密钥时页面自动切换至浏览器识别。
- 使用 `getUserMedia`、`AudioContext` 和 `AnalyserNode` 检测声音。
- 短于 2 秒的语音段使用 400ms 静音阈值，长语音段使用 600ms。
- 最短有效语音时长为 250ms，单段最长 10 秒。
- 使用 `MediaRecorder` 生成音频并发送至 `/api/transcribe`。
- 云端转写失败后进入“等待降级确认”，用户可通过语音或按钮选择浏览器识别或停止。

#### 浏览器模式

- 使用 Web Speech API，语言为 `zh-CN`，启用中间结果；单句结束后自动重启以保持持续聆听。
- 中间结果尝试解析并渲染半透明预览。
- 最终候选按“可解析性、命令关键词、ASR 置信度、短噪声惩罚”评分择优。
- 当前实现使用单句识别并自动重启、最多 10 个候选，并在浏览器支持时注入绘图领域词汇偏置。
- 稳定且本地规则可确定解析的中间结果约 400ms 后直接执行；未完成或无法确定的文本仍等待最终结果。

#### 唤醒词模式

- 点击主按钮会直接启动当前识别模式，不依赖唤醒词。
- 完整聆听中说“休息”或“停止聆听”后进入后台唤醒词监听。
- 先使用能量门控检测持续声音，再启动最长 3 秒的短时 Web Speech 识别。
- 支持“听画”“开始画”“嘿画布”。
- 唤醒成功后恢复用户已选择的识别模式并进入完整聆听；完整聆听中说“休息”或“停止聆听”可返回后台模式。
- 唤醒词依赖 Web Speech API，真实设备误触发率和长期稳定性尚未人工验收。

#### 离线模式

- UI、模式切换、模型下载进度、IndexedDB 缓存接口、VAD 音频路由和 mock 测试已经实现。
- 下载逻辑当前只把模型 ZIP 原始数据存入 IndexedDB，没有解压到 Vosk 虚拟文件系统。
- `_createRealRecognizer()` 仍回退到 mock；没有加载真实 Vosk WASM，也没有完成音频到 16kHz 单声道的可靠重采样。
- 因此“模型已缓存”目前不等于“真实离线识别可用”。

当前不存在一个全自动的“离线 → 云端 → 浏览器”统一降级链：离线模型不可用时可回到云端；云端失败时需要用户确认后切换浏览器识别。

### 2.5 指令处理管道

```text
识别候选
  └─ 浏览器模式候选项评分
       └─ FUZZY_MAP 短语级模糊纠错
            └─ 文本归一化与引号内容保护
                 ├─ 完整匹配复合模板：房子 / 雪人 / 笑脸 / 一排 / 一列
                 └─ 按标点和连接词分句
                      └─ 逐句本地规则解析
                           ├─ 成功：动作校验
                           └─ 失败：
                                ├─ 置信度低于 0.45：提示重说
                                └─ 其他情况：调用 /api/parse
                                     └─ 动作校验
                                          └─ 风险确认
                                               └─ 事务执行、渲染、播报
```

低置信度保护只拦截“低置信度且本地无法解析”的文本。明确可由本地规则解析的指令即使浏览器置信度低于 `0.45`，仍会执行，以避免 Web Speech 中文置信度不稳定导致正常指令失效。

## 3. 标准动作协议

### 3.1 动作类型

| 动作 | 主要字段 | 说明 |
| --- | --- | --- |
| `create` | `kind`，可选位置、尺寸和样式 | 创建八类图形之一 |
| `select` | `target` | 更新当前选择 |
| `update` | `target`, `changes` | 修改白名单属性或图层顺序 |
| `move` | `target`，位置或偏移量 | 绝对区域移动或相对移动 |
| `align` | `target`, `mode` | 六种对齐 |
| `distribute` | `target`, `axis` | 横向或纵向均匀分布 |
| `duplicate` / `delete` | `target` | 复制或删除目标 |
| `group` / `ungroup` | `target` | 记录或解除组合关系 |
| `history` | `operation` | `undo` 或 `redo`，必须单独执行 |
| `canvas` | `operation` | 清空或修改背景色 |
| `export` | `format` | 导出 SVG 或 PNG |
| `help` / `status` | 无 | 触发语音效果 |

### 3.2 图形、目标与属性

- 图形类型：`rect`、`circle`、`ellipse`、`triangle`、`star`、`line`、`arrow`、`text`。
- 目标可为 `selected`、`last`、`lastTwo`、`all`、对象 ID、自动名称、中文类型名或 ID 数组。
- 可更新属性：`fill`、`stroke`、`strokeWidth`、`opacity`、`rotation`、`text`、`width`、`height`、`zOrder`。
- 颜色动作值仅允许 `none` 或安全十六进制颜色，避免通过 SVG 属性注入任意内容。
- 一次请求必须包含 1 到 20 个动作；前后端都会拒绝未知字段、非法枚举、非法值域和历史动作混用。
- 清空动作必须携带 `requiresConfirmation: true`，但浏览器仍会对其他高风险动作统一追加确认。

### 3.3 风险确认

以下情况进入 8 秒确认窗口：

- 清空画布。
- 删除图形。
- 修改目标为 `all` 的图形。
- 一条指令包含三个及以上修改动作。

确认窗口内只接受“确认/是/好的”或“取消/否/不要”，其他指令不会穿透执行。

## 5. 服务端与安全边界

`main.py` 使用 `ThreadingHTTPServer` 和 `SimpleHTTPRequestHandler` 托管 `static/`，并提供两个 POST 接口：

| 接口 | 输入 | 输出 | 限制 |
| --- | --- | --- | --- |
| `/api/transcribe` | 原始音频请求体 | `{ "text": "..." }` | 最大 10MB；限制 WebM、Ogg、MP4、MP3、WAV |
| `/api/parse` | `{ text, context }` | `{ "actions": [...] }` | 文本最大 1000 字符；动作最多 20 个 |

配置策略：

- 应用只使用 `.env` 中的 `OPENAI_API_KEY`；加载器不会覆盖已有进程环境变量。
- `config.yaml` 保存 STT 和命令模型的 `base_url` 与 `model`。
- 两种外部服务共享同一个 `OPENAI_API_KEY`。
- 音频只在内存中转发，不写入本地磁盘。

安全策略：

- 模型动作在服务端和浏览器端分别校验。
- 动作类型、字段、枚举、数值范围、目标长度和颜色格式均使用白名单。
- 不支持代码执行、任意对象属性写入、任意 SVG 属性或绕过清空确认。
- 服务默认仅绑定 `127.0.0.1`。若使用 `--host 0.0.0.0` 暴露到网络，当前服务没有身份认证、速率限制或跨站请求防护，不适合作为公开服务。

## 7. 已知限制与技术债

1. **离线识别会被 UI 误认为可用**：缓存 ZIP 后 `checkModelAvailability()` 返回可用，识别器随后进入 mock 路径，但不会产生真实转写结果。
2. **离线音频链路未完成**：模型 ZIP 未解压，Vosk 虚拟文件系统未建立，真实识别器未初始化，音频也没有可靠重采样到 16kHz 单声道。
3. **降级状态机不是完整自动链**：云端失败后需要确认才切换浏览器；离线失败也没有统一经过云端再到浏览器。
4. **真实浏览器语音生命周期仍需人工验收**：已处理音频上下文挂起、播报失败恢复和模式切换状态保留，但不同 Chrome/Edge 版本的长期运行稳定性仍需真实设备验证。
5. **本地类型过滤选择缺失**：例如“选择所有圆形”当前会退化为选择全部图形或无法表达预期。
6. **组合不是一等对象**：选择某个组合成员不会自动让后续操作作用于整个组合。
7. **几何没有画布边界约束**：移动、缩放和重复排列可以把对象放到画布外；“一排/一列”数量较多时尤其明显。
8. **规则语法仍是精选覆盖**：复合模板要求整句匹配，复杂表达依赖 LLM，完整拼音纠错和完整中文数词语法未实现。
9. **工程文件不可恢复**：刷新页面会丢失画布状态；没有 JSON 工程格式、导入或自动保存。
10. **真实语音指标未建立**：当前没有真实设备准确率、误唤醒率和端到端延迟基线。

## 8. 下一步优先级

1. **明确离线能力状态**：在真实 Vosk 闭环前，将 UI 和 README 标记为实验性脚手架，避免把“模型已缓存”展示成“离线识别已就绪”。
2. **完成或移除真实 Vosk 路径**：实现 ZIP 解压、模型加载、16kHz 重采样、识别结果回调、失败降级和真实浏览器验收。
3. **建立真实语音验收矩阵**：覆盖云端、浏览器和离线模式的准确率、延迟、噪声、唤醒词与恢复行为。
4. **收敛语音状态机**：保存并恢复用户模式，定义统一的失败转换，补充模式切换和模型下载的语音入口。
5. **补齐本地规则边界**：增加按类型过滤选择、复合模板与后续子句混用、布局越界处理和更完整的数字表达。
6. **引入可编辑工程格式**：序列化状态、历史与组合关系，支持保存、加载和崩溃恢复。

## 9. 关键技术决策

| 决策 | 当前选择 | 原因 |
| --- | --- | --- |
| 指令理解 | 本地规则优先，LLM 处理长尾 | 高频路径确定、快速、可测试；模型不直接控制任意代码 |
| 批量动作表示 | 平铺动作数组，最多 20 个 | 校验简单，天然复用事务机制 |
| 状态提交 | 深拷贝执行后原子提交 | 任何子动作失败都不污染画布 |
| 默认语音模式 | 云端 STT | 当前中文准确率和实现完整度最高 |
| 浏览器识别定位 | 云端失败后的可选降级与唤醒词识别 | 无需服务端，但浏览器兼容性和置信度不稳定 |
| 离线识别定位 | 实验性可选能力 | 模型体积、性能和真实集成成本较高 |
| 解析线程 | 浏览器主线程 | 当前规则规模较小，测试中没有显示需要 Web Worker |
| 后端框架 | Python 标准库 | 部署简单、依赖少，适合本地单用户应用 |
| 安全边界 | 前后端双重动作白名单 | 防止模型输出扩大为任意执行能力 |
