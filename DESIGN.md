# 听画设计文档

> 当前实现基线：2026-06-12
> 本文以当前工作区代码和实际测试结果为准，用于区分已经闭环的能力、仅完成框架的能力和后续规划。

## 1. 产品定位与当前结论

听画是一款本地运行、以简体中文语音为主要输入方式的结构化矢量绘图工具。产品核心不是自由手绘或生成式图像，而是把自然语言转换为一组受约束、可校验、可撤销的绘图动作。

当前版本已经闭环的主链路为：

1. 浏览器采集语音，默认通过 OpenAI 兼容音频转写服务获得文本；同时支持百炼 `qwen3-asr-flash` 的 OpenAI 兼容音频消息接口。
2. 文本优先由本地确定性规则解析，未知表达可回退至 OpenAI 兼容命令模型。
3. 动作在浏览器状态副本中按事务执行，成功后统一提交并渲染 SVG。
4. 结果通过画面、状态文字和中文语音播报反馈。
5. 开放式绘本描述可由 `/api/interpret` 规划为受约束语义实体，再由本地可信模板渲染并继续修改。
6. 版本 3 艺术创作流程可生成三张差异构图小稿，选择或混合后进行锁定约束下的多轮审美精修，并在绘本、木刻、水墨独立渲染器之间切换。
7. 选稿后可请求受约束 PNG 纹理并缓存在 IndexedDB；纹理失败或丢失时完整矢量主体继续可用。

当前版本已经具备完整的结构化绘图、云端语音和自动测试覆盖。浏览器 Web Speech 识别与唤醒词已移除；真实 Vosk 离线识别尚未闭环，现有模块仅作为隔离实验脚手架保留。

### 1.1 产品目标

| 目标 | 当前状态 | 说明 |
| --- | --- | --- |
| 确定性结构化绘图 | 已完成 | 高频指令由本地规则稳定映射为标准动作 |
| 复杂指令可撤销 | 已完成 | 一段指令最多 20 个动作，作为单一事务提交 |
| 语音优先操作 | 基本完成 | 首次授权需要点击；当前仅提供云端识别 |
| 低延迟本地反馈 | 已完成框架 | 本地规则、状态更新和 SVG 渲染均在浏览器执行；真实设备指标待验收 |
| 云端识别与模型回退 | 已完成 | 密钥仅保存在 Python 服务端 |
| 完全离线语音识别 | 未完成 | Vosk WASM、模型解压加载和真实转写尚未实现 |

## 2. 系统架构

### 2.1 运行拓扑

```text
麦克风
  └─ AudioContext + VAD + MediaRecorder
       └─ POST /api/transcribe ── OpenAI 兼容 STT / 百炼 Qwen ASR
                         │
                         v
语音文本 ── 本地 parser.js ── 标准动作 ── model.js 事务执行 ── SVG 渲染
               │
               └─ 解析失败时 POST /api/interpret ── OpenAI 兼容命令模型
```

### 2.2 模块职责

| 模块 | 职责 | 依赖与边界 |
| --- | --- | --- |
| `static/model.js` | 状态模型、动作校验、目标解析、事务执行、历史记录、自动命名 | 纯 JavaScript，不依赖 DOM |
| `static/parser.js` | 中文数字、模糊纠错、文本归一化、分句、本地规则、复合图形拆解 | 输出标准动作，并调用浏览器侧动作校验 |
| `static/scene_schema.js` / `templates.js` | 语义实体白名单、受控参数与参数化绘本 SVG 模板 | 模型不能提供原始 SVG；实体整体选择 |
| `static/art_schema.js` / `renderers.js` | 版本 3 创作状态、三稿策略、审美精修、锁定与三风格独立渲染 | 三风格共享语义实体，不共享造型规则 |
| `static/texture_cache.js` / `server/art.py` | IndexedDB 纹理缓存与受约束构图、精修、OpenAI 兼容 PNG 纹理接口 | 拒绝外部 URL、SVG、超限尺寸和任意属性；失败保留矢量版本 |
| `static/app.js` | 云端语音状态机、VAD、预览与正式渲染、TTS、导出、模型回退、只读验收事件 | 浏览器编排层，依赖 DOM 和浏览器媒体 API |
| `static/acceptance.js` | `?acceptance=1` 引导式验收台、浏览器本地记录与报告导出 | 仅在验收模式动态加载，不保存音频 |
| `static/vosk_recognizer.js` | IndexedDB 模型缓存接口、离线识别器生命周期和 mock 注入 | 真实 Vosk WASM 初始化仍是桩代码 |
| `main.py` / `server/server.py` | 轻量入口、静态文件服务和进程生命周期 | 仅使用 Python 标准库 |
| `server/config.py` / `client.py` / `handler.py` / `schema.py` | 配置、云端客户端、API 路由、错误分类与服务端动作校验 | 密钥不进入浏览器；错误提供稳定分类 |
| `static/index.html` / `static/styles.css` | 画布、状态面板、云端重试控件、预览层和视觉反馈 | 无构建步骤 |

### 2.3 状态模型

核心状态由 `DrawingEngine` 管理：

```text
objects       图形数组，数组顺序同时表示 SVG 图层顺序
selection     当前选中的对象 ID
background    画布背景色
counters      各图形类型的自动命名计数器
lastCreated   最近一次创建或复制的对象 ID
nextId        下一个对象 ID
groups        groupId 到成员 ID 数组的映射
nextGroupId   下一个组合 ID
scene         主题、情绪、构图说明、摘要与忽略项
art           创作意图、构图、艺术指导、锁定、小稿、阶段与纹理元数据
```

每次修改动作都先作用于深拷贝状态。任意动作失败时不提交；全部成功后只生成一条撤销历史。撤销栈最多保留 50 条，执行新事务后清空重做栈。

`export`、`help`、`status` 是效果动作，不修改状态；`history` 必须单独执行。

### 2.4 云端语音状态转换

- 当前唯一正式识别模式。
- 服务端状态接口会检查云端转写配置；未配置密钥时显示明确错误。
- 使用 `getUserMedia`、`AudioContext` 和 `AnalyserNode` 检测声音。
- 短于 2 秒的语音段使用 400ms 静音阈值，长语音段使用 600ms。
- 最短有效语音时长为 250ms，单段最长 10 秒。
- 使用 `MediaRecorder` 生成音频并发送至 `/api/transcribe`。
- 监测麦克风轨道结束、录音器异常停止、`AudioContext` 挂起和页面恢复；非用户主动停止时自动恢复一次，再次失败后进入手动重试。
- 云端转写失败后显示“重试云端识别”和“停止聆听”；重试会重新检查服务配置并恢复录音。
- “休息”和“停止聆听”均直接停止识别，不进入后台监听。

#### 离线实验模块

- IndexedDB 缓存接口、VAD 音频路由和 mock 测试脚手架保留。
- 下载逻辑当前只把模型 ZIP 原始数据存入 IndexedDB，没有解压到 Vosk 虚拟文件系统。
- `_createRealRecognizer()` 仍回退到 mock；没有加载真实 Vosk WASM，也没有完成音频到 16kHz 单声道的可靠重采样。
- 因此“模型已缓存”目前不等于“真实离线识别可用”。

第二版普通用户只可使用云端识别；浏览器识别和后台唤醒词不再属于产品能力。

### 2.5 指令处理管道

```text
云端转写文本
  └─ FUZZY_MAP 短语级模糊纠错
            └─ 文本归一化与引号内容保护
                 ├─ 完整匹配复合模板：房子 / 雪人 / 笑脸 / 一排 / 一列
                 └─ 按标点和连接词分句
                      └─ 逐句本地规则解析
                           ├─ 成功：动作校验
                           └─ 失败：
                                ├─ 置信度低于 0.45：提示重说
                                └─ 其他情况：调用 /api/interpret
                                     └─ 动作校验
                                          └─ 事务执行、渲染、播报
```

低置信度保护接口仍保留给文本处理入口；当前云端转写不提供浏览器候选置信度。

## 3. 标准动作协议

### 3.1 动作类型

| 动作 | 主要字段 | 说明 |
| --- | --- | --- |
| `create` | `kind`，可选位置、尺寸和样式 | 创建八类图形之一 |
| `select` | `target` | 更新当前选择 |
| `update` | `target`, `changes` | 修改白名单属性或图层顺序 |
| `move` | `target`，位置或偏移量 | 绝对区域移动或相对移动 |
| `align` | `target`, `mode` | 六种对齐 |
| `distribute` | `target`, `axis` | 横向或纵向均匀分布 |
| `duplicate` / `delete` | `target` | 复制或删除目标 |
| `group` / `ungroup` | `target` | 记录或解除组合关系 |
| `history` | `operation` | `undo` 或 `redo`，必须单独执行 |
| `canvas` | `operation` | 清空或修改背景色 |
| `export` | `format` | 导出 SVG 或 PNG |
| `help` / `status` | 无 | 触发语音效果 |
| `entity_create` / `entity_update` | 模板、名称、边界、层次、受控参数 | 创建或修改整体语义实体 |
| `scene_update` | 主题、情绪、构图、摘要、忽略项 | 更新场景语义元数据 |

### 3.2 图形、目标与属性

- 图形类型：`rect`、`circle`、`ellipse`、`triangle`、`star`、`line`、`arrow`、`text`。
- 目标可为 `selected`、`last`、`lastTwo`、`all`、对象 ID、自动名称、中文类型名或 ID 数组。
- 可更新属性：`fill`、`stroke`、`strokeWidth`、`opacity`、`rotation`、`text`、`width`、`height`、`zOrder`。
- 颜色动作值仅允许 `none` 或安全十六进制颜色，避免通过 SVG 属性注入任意内容。
- 一次请求必须包含 1 到 20 个动作；前后端都会拒绝未知字段、非法枚举、非法值域和历史动作混用。

## 5. 服务端与安全边界

`main.py` 委托 `server/` 模块使用 `ThreadingHTTPServer` 和 `SimpleHTTPRequestHandler` 托管 `static/`，并提供两个 POST 接口：

| 接口 | 输入 | 输出 | 限制 |
| --- | --- | --- | --- |
| `/api/transcribe` | 原始音频请求体 | `{ "text": "..." }` 或结构化错误 | 最大 10MB；限制 WebM、Ogg、MP4、MP3、WAV |
| `/api/parse` | `{ text, context }` | `{ "actions": [...] }` 或结构化错误 | 旧标准动作兼容接口 |
| `/api/interpret` | `{ text, context }` | `actions`、`scene_plan` 或 `scene_revision` | 最多 20 个实体；拒绝未知模板、原始 SVG 和非法参数 |

错误响应保留可读的 `error` 文本，并增加稳定的 `errorCode` 与布尔值 `retryable`。分类覆盖配置缺失、认证、权限、模型不存在、限流、网络、超时、无效响应和空转写。

配置策略：

- 应用使用 `.env` 中的 `OPENAI_API_KEY`，纹理服务可选使用独立的 `TEXTURE_API_KEY`；加载器不会覆盖已有进程环境变量。
- `config.yaml` 保存 STT、命令模型和可选纹理模型的 `base_url` 与 `model`。
- 纹理服务调用 OpenAI 兼容 `/images/generations` 接口，只请求 `b64_json`；没有独立纹理密钥时共享 `OPENAI_API_KEY`。
- 音频只在内存中转发，不写入本地磁盘。

安全策略：

- 模型动作在服务端和浏览器端分别校验。
- 动作类型、字段、枚举、数值范围、目标长度和颜色格式均使用白名单。
- 不支持代码执行、任意对象属性写入或任意 SVG 属性。
- 云端纹理响应必须是内联 PNG，且不超过 2048 像素和 5MB；外部 URL、SVG 或无效响应会显式失败并回退到可编辑矢量版本。
- 服务默认仅绑定 `127.0.0.1`。若使用 `--host 0.0.0.0` 暴露到网络，当前服务没有身份认证、速率限制或跨站请求防护，不适合作为公开服务。

## 7. 已知限制与技术债

1. **离线识别仅是隔离脚手架**：`vosk_recognizer.js` 仍包含缓存和 mock 路径，但正式应用不加载或暴露该模块。
2. **离线音频链路未完成**：模型 ZIP 未解压，Vosk 虚拟文件系统未建立，真实识别器未初始化，音频也没有可靠重采样到 16kHz 单声道。
3. **云端识别是单点依赖**：云端 STT 不可用时只能重试或停止，没有其他正式识别路径。
4. **真实浏览器语音生命周期仍需人工验收**：已实现轨道、录音器、音频上下文和页面恢复异常的单次自动恢复，并处理播报暂停与异步停止竞态，但长期运行稳定性仍需真实设备验证。
5. **一般几何操作允许越界**：批量“一排/一列”已经限制在画布内，但自由移动和缩放仍可把对象放到画布外。
6. **规则语法仍是精选覆盖**：复杂表达依赖 LLM，完整拼音纠错和完整中文数词语法未实现。
7. **真实语音指标尚待人工采集**：验收台会记录会话时长、中断原因和自动恢复结果，但 Chrome/Edge 真实麦克风准确率、延迟和 30 分钟稳定性仍未执行。

## 8. 下一步优先级

1. **保持离线能力隔离**：在真实 Vosk 闭环前，不把实验模块重新接入正式应用。
2. **完成 Chrome/Edge 人工发布验收**：使用引导式验收台采集云端准确率、延迟、重试、TTS 恢复、导出和 30 分钟稳定性证据。
3. **根据真实指标调优 VAD 与提示**：只依据真实验收报告调整阈值和错误恢复体验。
4. **完成或移除真实 Vosk 路径**：实现 ZIP 解压、模型加载、16kHz 重采样、识别结果回调和真实浏览器验收，或删除实验脚手架。
5. **继续补齐规则长尾**：扩展复杂表达、拼音纠错和更完整的中文数词语法。

## 9. 关键技术决策

| 决策 | 当前选择 | 原因 |
| --- | --- | --- |
| 指令理解 | 本地规则优先，LLM 处理长尾 | 高频路径确定、快速、可测试；模型不直接控制任意代码 |
| 批量动作表示 | 平铺动作数组，最多 20 个 | 校验简单，天然复用事务机制 |
| 状态提交 | 深拷贝执行后原子提交 | 任何子动作失败都不污染画布 |
| 正式语音识别 | 仅云端 STT | 中文准确率和实现完整度最高；避免暴露不可用的 Web Speech 路径 |
| 离线识别定位 | 隔离实验模块 | 模型体积、性能和真实集成成本较高 |
| 解析线程 | 浏览器主线程 | 当前规则规模较小，测试中没有显示需要 Web Worker |
| 后端框架 | Python 标准库 | 部署简单、依赖少，适合本地单用户应用 |
| 安全边界 | 前后端双重动作白名单 | 防止模型输出扩大为任意执行能力 |
