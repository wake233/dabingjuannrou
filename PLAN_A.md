# 听画全面绘画能力升级计划

## 1. 总结

保留语义可编辑 SVG 核心，将项目升级为工程版本 4。引入开源几何、笔触、曲线、色彩与程序纹理工具，建立统一艺术渲染引擎；优先把现有 12 个旗舰题材的绘本风格做到作品级，木刻与水墨接入共享造型能力。

本地绘画、素材与导出必须完全离线可用。模型负责自由规划，本地艺术导演负责校正构图和视觉质量。

## 2. 开源工具链

通过 `npm + esbuild` 锁定、打包依赖，只向现有 SVG 渲染器暴露受控适配接口：

- `perfect-freehand`：生成自然、有压感和收笔变化的实心笔触。
- `D3-shape`：生成有机曲线、轮廓和区域路径。
- `Bezier.js`：处理贝塞尔曲线采样、切线、偏移与交点。
- `Culori`：使用感知色彩空间生成协调色板、明暗与冷暖关系。
- `simplex-noise.js`：生成确定性纸张、墨迹、材质与自然轮廓扰动。
- `esbuild`：构建浏览器端 ESM 工具包。
- 使用 Open Peeps、Humaaans 等 CC0 素材作为人物结构参考和可拆解部件来源；所有采用资产统一重绘并记录来源。

不使用 Paper.js、Fabric.js、Two.js 替换现有状态模型和 SVG 主画布；不使用 Rough.js 作为精绘主渲染器。

## 3. 实施变更

### 3.1 艺术引擎基础

- 新建独立艺术引擎层，包含几何、笔触、形状语法、色彩、光照、材质、构图评估和资产注册模块。
- 将第三方库封装为确定性接口；所有随机效果必须由工程种子驱动。
- 把当前单文件模板重构为人物动物、建筑器物、植物自然、天气光效四类形状语法。
- 建立统一渲染流程：语义骨架 → 有机轮廓 → 材质块面 → 结构线 → 光影 → 纹理细节 → 风格解释。
- 绘本使用完整流程；木刻和水墨复用语义骨架与轮廓，再应用独立块面、刻线、墨层和留白规则。

### 3.2 工程版本 4

新增可保存的 `art.renderProfile`：

```js
{
  seed: string,
  camera: "wide" | "medium" | "portrait",
  lighting: "soft-day" | "golden-hour" | "night" | "rain",
  material: "paper" | "smooth" | "carved" | "ink-wash",
  detail: "draft" | "balanced" | "gallery"
}
```

- 新增受校验的 `creative/set_render_profile` 动作，用于语音修改镜头、光线、材质和细节等级。
- 扩展锁定字段，允许锁定 `camera`、`light`、`material` 和 `detail`。
- 将版本 1–3 工程迁移到版本 4；默认种子由场景内容稳定生成，旧工程视觉允许升级但语义状态保持不变。

### 3.3 本地艺术导演

- 模型继续返回受约束的场景规划，不允许输出 SVG 或路径。
- 在模型规划后运行本地艺术导演，对焦点尺度、遮挡、负空间、前中后景、色板和光源进行确定性校正。
- 为 12 个旗舰题材建立艺术指导配置，定义主体角色、推荐构图范围、深度层级、色彩关系和必要叙事元素。
- 不合格方案自动校正一次；仍违反主体过小、严重遮挡、元素堆叠或缺少层次等硬性规则时拒绝展示。
- 普通开放题材使用通用艺术导演规则，不强行套用旗舰题材模板。

### 3.4 旗舰题材与素材

- 重点升级作品集中的 12 个题材，每题材至少支持两种明显不同的有效构图。
- 优先重做人物、猫狗鸟、树与花草、房屋建筑、街道河流、天气和光源等高复用实体。
- 素材仅接受 CC0 或 MIT；加入资产清单，记录名称、来源、许可证、修改说明和使用模块。
- 所有第三方素材随构建产物本地提供，运行时禁止请求外部素材、字体或脚本。

## 4. 测试与验收计划

- 验证开源工具适配器的确定性、有限坐标、节点上限、曲线连续性、笔触起收和纹理种子。
- 验证版本 1–3 到版本 4 的迁移、保存、恢复、撤销、锁定与导出。
- 为艺术导演建立自动测试：主体尺度、碰撞、遮挡、深度层级、负空间、色板协调和光线一致性。
- 为 12 个旗舰题材建立结构化质量评分，每题材两种构图必须达到阈值。
- 浏览器自动验证主画布、SVG、PNG 与作品集渲染一致，并确认无外部网络依赖。
- 性能门槛保持：20 个实体基础渲染低于 `100ms`，完整精绘在基础画面后 `500ms` 内完成。
- 自动检查第三方许可证和资产来源清单。
- 自动指标作为主要完成标准；同时保存浏览器截图供人工抽检，但人工评分不阻塞第一轮完成。
- 每阶段运行 Node、Python、smoke、浏览器、导出、离线网络和 `git diff --check` 验证，并创建独立 Git commit。

## 5. 交付阶段

1. 建立构建流程、开源工具适配层、许可证清单和版本 4 数据结构。
2. 完成共享几何、笔触、色彩、光照、材质与本地艺术导演。
3. 重做高复用实体并完成前六个旗舰题材。
4. 完成剩余六个旗舰题材，并让木刻、水墨接入共享造型系统。
5. 完成自动质量门槛、离线验证、性能优化、迁移和验收报告。

## 6. 假设与约束

- 核心输出保持为可编辑 SVG，可选质感层必须可移除。
- 绘本为第一轮作品级主风格；木刻与水墨先完成共享造型能力接入和基础风格差异。
- 模型继续自由规划，最终场景必须通过本地艺术导演校正。
- 安装依赖并完成构建后，本地绘画能力必须完全离线可用。
- 实施时保留并合并当前工作区已有修改，不覆盖用户未提交内容。
