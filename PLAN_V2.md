## 1. 当前基线与第二版目标

当前版本已经具备结构化矢量绘图、规则优先解析、模型回退、事务撤销、风险确认、云端语音、浏览器识别降级、唤醒词框架以及 SVG/PNG 导出。

第二版的目标是先闭环现有能力的正确性、安全边界和可恢复性，再扩展本地规则与工程格式。第二版不交付真实 Vosk 离线识别；现有 Vosk 代码仅作为隔离的实验脚手架保留，不向普通用户展示不可用入口。

### 第二版范围原则

- 优先修复会产生错误画面、绕过确认、错误导出或阻塞服务的缺陷。
- 云端识别失败后保留用户确认，再切换到浏览器识别；不实施静默自动降级。
- 标准动作协议保持简单，不为类型选择增加新动作类型或对象型目标。
- 离线识别、绝对坐标、修饰语剥离和大规模 LLM 提示词扩展不进入第二版交付范围。
- 自动测试必须验证动作执行后的真实状态，不能只检查解析得到的动作数组。

## 2. P0：发布阻断缺陷

P0 必须在任何第二版功能扩展前完成。

### 2.1 复合指令语义闭环

**现状**：复合模板已经可以和后续子句一起解析，但每个 `create` 都会覆盖 `lastCreated`。因此“画一个房子，然后移动到右边”只移动最后创建的门；“画一排三个矩形，然后顶部对齐”会因目标不足导致整段事务回滚。

**实现要求**

- 同一 `_compositeId` 下连续创建的对象共同写入 `lastCreated`。
- 复合创建结束后，`last`、后续“它”和默认上下文引用解析为该复合结果的全部成员。
- 普通连续创建仍保持现有语义，不应被错误合并为一个复合结果。
- 将“复合模板与后续子句混用”的状态从完成改为部分完成，直到执行级测试全部通过。

**验收标准**

- “画一个房子，然后移动到右边”移动房子的全部组件。
- “画一个雪人，然后复制它”复制雪人的全部组件。
- “画一排三个矩形，然后顶部对齐”事务成功，三个矩形均保留。
- 上述每条指令仍只生成一条撤销历史。

### 2.2 收紧内部元数据边界

**现状**：前后端校验会放行任意以下划线开头的字段；风险确认又信任 `_compositeId`，模型响应可能伪造该字段并绕过多动作确认。

**实现要求**

- 服务端模型动作拒绝所有 `_` 前缀字段。
- 浏览器动作校验只允许 `_compositeId`，且值必须为正整数。
- `_compositeId` 只能由本地解析器在复合拆解阶段生成。
- 风险确认继续把同一复合组视为一个语义单元。
- 前后端动作白名单测试明确覆盖元数据差异。

**验收标准**

- 模型返回含 `_compositeId` 或其他 `_` 字段时，服务端拒绝响应。
- 浏览器拒绝非法、非整数或未知内部元数据。
- 本地复合模板仍能执行，且不会因多个组件误触发大型事务确认。

### 2.3 修复导出与动作边界

**实现要求**

- SVG 和 PNG 从不含选择高亮、预览层和临时交互效果的画布副本生成。
- 创建图形时使用空值判断保存 `strokeWidth`，合法值 `0` 不得回退为默认值。
- `/api/parse` 拒绝缺失、零或负数 `Content-Length`；请求体上限保持 100KB。

**验收标准**

- 选中图形后导出的 SVG/PNG 不含 `selection-glow`。
- `{ strokeWidth: 0 }` 创建的图形线宽保持为 `0`。
- `/api/parse` 对无效 `Content-Length` 快速返回错误，不等待连接结束。

### 2.4 隐藏不可用离线入口

**实现要求**

- 普通模式切换只展示云端和浏览器识别。
- 隐藏离线模型下载按钮，不提供离线下载或切换的语音入口。
- 保留 `vosk_recognizer.js`、mock 测试和实验代码，避免影响未来验证。
- README 与 DESIGN 明确说明真实离线识别不属于第二版交付范围。

**验收标准**

- 普通用户无法从页面或语音进入不可用的离线模式。
- 云端和浏览器模式行为不受影响。
- Vosk 实验模块测试继续通过。

## 3. P1：语音状态机收敛

P1 在 P0 完成后实施，可与 P2 并行。

### 3.1 统一失败提示与确认降级

**实现要求**

- 保留云端失败后的确认面板，用户确认后才切换浏览器识别。
- 云端 API 不可用、浏览器不支持、权限拒绝和网络错误使用一致的状态提示。
- 确认面板只提供当前真实可用的选择；浏览器不可用时只能停止或重试云端。
- 模式切换、TTS 暂停和识别恢复不得产生重复监听实例。

**验收标准**

- 云端失败后不会静默切换模式。
- 用户确认使用浏览器识别后恢复持续聆听。
- 浏览器不可用时不会展示不可执行的降级选项。

### 3.2 保存并恢复用户模式

**实现要求**

- 仅持久化 `cloud` 或 `browser` 用户选择。
- 页面加载和唤醒恢复时优先使用保存模式。
- 保存模式不可用时回退云端并提示用户；若云端也不可用，再进入现有确认降级流程。

**验收标准**

- 用户切换到浏览器模式并刷新后，模式仍为浏览器识别。
- 从后台唤醒后恢复用户保存的可用模式。
- 无效或旧版离线模式存储值不会令应用进入不可用状态。

### 3.3 模式管理语音入口

**实现要求**

- 支持“切换到云端识别”“使用云端识别”。
- 支持“切换到浏览器识别”“使用浏览器识别”。
- 支持“当前识别模式”“现在是什么模式”。
- 语音切换复用现有模式切换状态机，并通过 TTS 明确反馈结果。

**验收标准**

- 两种可用模式均可通过语音切换。
- 查询模式时只播报状态，不修改画布或识别模式。
- 不增加离线模式或模型下载的语音入口。

## 4. P2：本地规则与协议补齐

### 4.1 按类型过滤选择

**实现要求**

- “选择所有圆形”“选择全部矩形”等指令解析为现有中文类型名字符串目标。
- 复用状态模型已有的中文类型目标解析，不新增对象型 `target` 或新动作类型。
- “选择所有图形”仍解析为 `all`，不得被类型规则截获。

**验收标准**

- 画布包含三圆两矩形时，“选择所有圆形”只选择三个圆形。
- 类型不存在时事务失败且当前选择保持不变。
- 前后端动作协议无需新增字段。

### 4.2 排列边界约束

**实现要求**

- “一排/一列”复合模板在 1000×700 画布内计算尺寸和间距。
- 优先缩小间距，仍无法容纳时按比例缩小图形。
- 所有生成对象必须完整位于画布内。
- 不向标准动作协议增加 `overflow` 等临时字段。

**验收标准**

- 一排十个圆和一列十个矩形全部位于画布内。
- 数量达到动作上限时仍不会产生负尺寸或越界坐标。
- 排列结果保持等间距。

### 4.3 扩展中文数字

**实现要求**

- 支持自然的十、百、千位组合，包括“一百二十”“两百五”“三千零六”。
- 保持阿拉伯数字、小数和现有简单中文数字行为。
- 数量、尺寸和移动距离继续受动作协议值域及 20 动作上限约束。

**验收标准**

- 数字转换覆盖自然中文表达及非法输入。
- 超过复合动作上限的数量被明确拒绝，不截断执行。
- 现有解析测试无回归。

### 4.4 前后端动作验证一致性

**实现要求**

- 为每类动作建立共享测试向量，分别运行浏览器侧和服务端校验。
- 覆盖字段白名单、颜色、数值边界、目标、历史动作混用、清空确认和内部元数据。
- 保持服务端模型输入比浏览器本地动作更严格。

**验收标准**

- 公共标准动作在前后端均接受。
- 非内部非法动作在前后端均拒绝。
- 仅本地 `_compositeId` 的差异有明确测试和注释。

## 5. P3：组合与工程格式

P3 在 P0、P2.1 完成后实施。

### 5.1 组合升级为一等可命名对象

**数据结构**

```json
{
  "group-1": {
    "id": "group-1",
    "name": "组合一",
    "members": ["shape-1", "shape-2"]
  }
}
```

**实现要求**

- `selection` 继续保存成员对象 ID，不保存组合 ID。
- 按组合名称解析目标时展开为全部有效成员。
- 删除或重新组合成员时同步更新组合；成员不足两个时自动解散。
- 组合名称使用稳定计数器生成，并纳入撤销、重做和工程序列化。

**验收标准**

- 创建组合后可通过“组合一”选择、移动、复制和删除全部成员。
- 撤销与重做完整恢复组合名称和成员关系。
- 成员删除后不会留下无效组合引用。

### 5.2 带版本工程格式

**文件格式**

```json
{
  "format": "listen-paint",
  "version": 1,
  "state": {},
  "history": {
    "undo": [],
    "redo": []
  }
}
```

**实现要求**

- 保存完整状态、组合关系、计数器和历史。
- 加载前校验格式标识、版本、对象结构、ID 唯一性、组合引用和全部属性值域。
- 加载使用临时引擎验证；任何错误都不得修改当前画布。
- 不执行工程文件中的代码、URL 或任意 SVG 属性。

**验收标准**

- 保存后加载可恢复图形、背景、选择、组合和撤销历史。
- 损坏、未知版本或非法属性的工程文件被拒绝，当前画布保持不变。
- 工程格式测试包含往返序列化和恶意载荷。

### 5.3 自动保存与崩溃恢复

**实现要求**

- 仅在事务成功提交、撤销、重做或工程成功加载后更新自动保存。
- 页面启动时校验自动保存内容，再恢复到临时引擎。
- 无效自动保存被忽略并提示，不覆盖空白初始状态。
- 提供“丢弃上次工程”语音入口。

**验收标准**

- 刷新页面可恢复最近一次成功提交的状态。
- 失败事务不会覆盖可恢复状态。
- 损坏自动保存不会阻止应用启动。

## 6. P4：真实环境验收

第二版真实验收只覆盖云端和浏览器模式。

### 6.1 验收矩阵与执行记录

建立 `docs/voice-acceptance.md`，记录：

- Chrome 和 Edge 版本、操作系统、麦克风、网络条件。
- 20 条标准指令在云端和浏览器模式下的转写、解析、执行结果。
- 语音结束到画面更新的端到端延迟。
- 首次授权、TTS 暂停恢复、确认降级、唤醒词和连续使用稳定性。
- SVG/PNG 在不同查看器中的结果，确认不存在选择高亮或预览内容。

### 6.2 第二版验收标准

- 云端模式标准指令准确执行率不低于 90%。
- 浏览器模式标准指令准确执行率不低于 70%。
- 本地规则命令从最终文本到执行完成的响应时间目标低于 100ms。
- 简单语音指令端到端延迟目标低于 1 秒。
- 安静环境唤醒词 30 分钟无误触发；唤醒响应低于 2 秒。
- 连续使用 30 分钟无崩溃、重复识别实例或不可恢复中断。

## 7. 依赖与执行顺序

```text
P0 发布阻断缺陷
 ├─ 复合语义闭环
 ├─ 内部元数据边界
 ├─ 导出与动作边界
 └─ 隐藏离线入口
        │
        ├─ P1 语音状态机
        └─ P2 本地规则与验证
              │
              └─ P3 组合与工程格式
                    │
                    └─ P4 真实环境验收
```

- P0 内部任务可并行，但必须整体完成后再判定第二版主链路稳定。
- P1 与 P2 可并行。
- P3 依赖复合语义稳定和类型选择协议确定。
- P4 验收模板可提前创建，最终执行必须基于候选发布版本。

## 8. 第二版发布门槛

- P0 至 P3 全部完成，P4 验收结果达到标准。
- `node --test tests/*.test.js` 与 Python `unittest` 全部通过。
- 自动测试覆盖复合动作真实执行、内部元数据伪造、零线宽、无效 `Content-Length`、无高亮导出和损坏工程加载。
- Chrome/Edge 完成云端、浏览器、风险确认、唤醒恢复、工程恢复和导出人工验收。
- README、DESIGN 和 PLAN 对离线能力、降级行为和已完成能力的描述一致。
- 无已知可绕过风险确认、污染事务状态或导致当前画布丢失的缺陷。

## 9. 后续版本候选项

- 完成或移除真实 Vosk：ZIP 解压、WASM 加载、16kHz 重采样、断网识别和真实设备验收。
- 离线模型下载、删除和模式切换的语音入口。
- 绝对坐标和更高级的空间表达。
- 本地复合模板修饰语剥离与参数化。
- 扩展 LLM 提示词中的几何修饰语、空间关系和长尾语义概念。
- 自由手绘、图片导入和生成式图像。
