# 听画本地候选版发布收口计划

## 1. 总结

将当前第二版整理为可发布的本地候选版。先独立提交当前已验证改动，再统一设计文档、完善云端语音错误分类与指标采集、增加引导式真实语音验收台，最后完成 Chrome/Edge 人工验收。

当前基线：Node `93/93`、Python `25/25`、smoke `12/12` 通过。

## 2. 实施变更

### 2.1 固化当前基线

- 审查当前未提交改动，排除确认无关的文件后，将现有第二版实现及 `server/` 重构作为独立提交。
- 提交前重跑全量测试、smoke 和 `git diff --check`。
- 后续发布收口改动使用独立提交，不混入该基线提交。

### 2.2 统一云端错误与指标

- 服务端 API 保留现有 `error` 文本，并增加稳定的 `errorCode` 与 `retryable` 字段，保持向后兼容。
- 统一分类配置缺失、认证失败、权限不足、模型不存在、限流、网络失败、超时、无效响应和空转写。
- 浏览器侧统一麦克风权限、浏览器能力、API 不可达和云端服务错误的状态提示。
- 记录语音段提交、转写完成、命令执行完成、本地处理耗时和失败类别；不记录或保存音频。

### 2.3 引导式验收台

- 保留诊断页作为环境预检，并增加进入验收模式的入口。
- 通过主应用的 `?acceptance=1` 模式加载独立验收面板，避免复制语音与绘图链路。
- 验收面板逐条展示云端模式标准指令，自动记录实际转写、执行成功与否、本地处理耗时、端到端耗时和错误分类。
- 用户可修正通过/失败判断、填写环境信息，并导出 JSON 与 Markdown 报告。
- 验收数据仅保存在当前浏览器本地，提供明确的清除入口。

### 2.4 文档事实收敛

- 修正 `DESIGN.md` 中已经完成但仍被列为技术债的类型选择、组合、布局边界和工程恢复内容。
- 更新服务端模块职责、实际测试数量和下一阶段优先级。
- 将真实验收矩阵改为仅覆盖云端模式，删除已移除的浏览器识别与唤醒词要求。
- 修正 README 中仍要求清空确认等与当前实现不一致的描述。
- 将 `PLAN_V2.md` 标记为历史计划，发布收口计划成为当前执行依据。

## 3. 公共接口

- `/api/parse` 与 `/api/transcribe` 错误响应新增：
  - `errorCode`：稳定错误类别。
  - `retryable`：是否适合直接重试。
  - 现有 `error` 字符串继续保留。
- 浏览器新增只读验收事件：语音段提交、转写完成、命令完成和错误。
- `?acceptance=1` 启用验收面板；普通应用入口行为保持不变。

## 4. 测试与验收计划

- 自动测试覆盖服务端错误分类、重试属性、浏览器错误提示、指标事件顺序、验收记录持久化、报告导出和清除。
- 验证普通模式不会显示验收面板，也不会改变绘图、语音或自动保存行为。
- 每个实施阶段运行 Node、Python、smoke 和 `git diff --check`，完成后创建独立 Git commit。
- Chrome 与 Edge 分别完成云端 20 条指令验收，至少 `18/20` 正确执行。
- 两个浏览器分别验证首次授权、TTS 暂停恢复、云端失败重试、30 分钟连续使用和 SVG/PNG 导出。
- 本地规则处理耗时目标低于 `100ms`；简单语音端到端耗时目标低于 `1s`。
- 在两份浏览器验收报告达到门槛前，候选版状态保持“人工验收阻塞”，不得标记完整发布通过。

## 5. 假设与默认选择

- 发布目标仅为本地单用户候选版，不增加公网身份认证、限流或跨站防护。
- 真实 Vosk 离线识别继续隔离，不在本阶段接入正式应用。
- 不保存用户音频；验收报告仅包含环境、指令、转写文本、结果、耗时和错误类别。
- 当前未提交改动经审查后作为独立基线提交。
